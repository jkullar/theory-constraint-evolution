# 02_models — Index

This folder contains **models**: reusable frameworks that explain *how* the Constraint Evolution Theory works mechanically. A model is not a historical fact or a prediction — it is an abstraction that captures a repeating pattern you can apply across different ages and contexts.

## What is a Model?

A model is a simplified structure that:
- Describes a **mechanism** (how something works, not just what happened)
- Is **reusable** across ages, domains, or contexts
- Makes the theory **testable** by defining what you would expect to see if it were true
- Can be **wrong** — and that is useful, because it forces refinement

Models live here. Evidence and examples live in `/03_timeline`. Predictions generated from models live in `/04_future`.
