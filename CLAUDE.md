# CLAUDE.md — Constraint Evolution Theory

## What This Is

A version-controlled thinking system for developing the **Constraint Evolution Theory** — the idea that human progress follows a repeating loop:

> Scarcity → Build → Scale → Coordination → New Scarcity

Ideas are treated like code: refined in commits, explored in branches, stabilized via merges.

---

## Core Concepts

| Primitive | Role |
|---|---|
| Scarcity | Defines each age — what is lacking |
| Builders | Create systems that solve scarcity |
| Coordinators / People Movers | Coordinate humans at scale |
| Coordination | The recurring bottleneck after scale |

**Timeline:** Survival → Agriculture → Industrial → Information → Alignment

## My Role as AI

Defined in `theory-of-constraint-evolution/AI_GUIDE.md`. In short:

1. **Clarify** — simplify, remove ambiguity, improve structure
2. **Detect problems** — contradictions, weak assumptions, missing links
3. **Propose improvements** — better models, refactored ideas, extensions
4. **Maintain consistency** — concepts must align across files

> Goal: improve the **quality of thinking**, not generate content for its own sake.

## Commit Convention

```
feat:     new idea
refactor: restructure existing idea
fix:      corrected mistake
add:      added clarity or example
drop:     removed incorrect idea
```

Each commit = one atomic idea change.

---

## Branch Naming

```
exploration/*   → new ideas

future/*        → predictions
applications/*  → opportunities
```

## Output Format Rules

- Use clear headings
- Be concise — clarity over complexity
- Structure responses logically
- One idea per change — do not mix concepts
- Challenge ideas — do not assume correctness
