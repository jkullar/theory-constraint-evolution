# How to Use This Repository

## 🧠 Core Philosophy

This repository treats ideas like code.

* Ideas are not static
* Understanding evolves
* Every refinement is tracked

> Every commit = a change in understanding

---

# 🔁 The Thinking Loop

1. Identify a gap / question
2. Explore current understanding
3. Refine or restructure
4. Commit the change
5. Repeat

---

# 🧩 Repository Structure

* `/timeline` → ages and constraints
* `/primitives` → core concepts (scarcity, builders, coordination)
* `/models` → reusable frameworks
* `/future` → predictions
* `/ideas` → opportunities

---

# ⚙️ Git Features as Thinking Tools

## 🏷️ Labels (VERY IMPORTANT)

Labels classify thinking work.

### Types:

**Concept**

* primitive
* model
* definition

**Exploration**

* hypothesis
* open-question
* contradiction

**Refinement**

* needs-clarity
* refactor-theory

**Application**

* opportunity
* startup-idea
* prediction

---

## 🐛 Issues → Thought Units

Each issue = one thinking problem.

Use issues for:

* unclear ideas
* contradictions
* new hypotheses

### Structure:

* Problem
* Current Thinking
* Doubts
* Next Step

---

## 🔀 Branches → Thinking Modes

Branches allow safe exploration.

### Naming:

* exploration/* → new ideas
* refactor/* → restructuring
* future/* → predictions
* applications/* → opportunities

Merge only when:

* idea improves clarity
* or survives critique

---

## 🔧 Commits → Atomic Idea Changes

Each commit must represent **one clear idea change**.

### Format:

* feat: new idea
* refactor: restructure
* fix: corrected mistake
* add: added clarity/example
* drop: removed incorrect idea

### Examples:

* feat: define alignment as system constraint
* fix: attention is not universal scarcity
* refactor: separate builders vs coordinators

---

## 🔀 Pull Requests → Idea Upgrades

PRs represent **proposed improvements to the theory**.

Each PR must answer:

* What changed?
* Why?
* What does it affect?
* What remains uncertain?

---

## 🏁 Milestones → Theory Versions

Milestones track evolution of the theory.

### Example:

* v0.1 → raw structure
* v0.2 → scarcity timeline defined
* v0.3 → builder/coordinator model
* v1.0 → publishable theory

---

## 🧾 CHANGELOG.md → Evolution Log (Optional but Powerful)

Tracks major shifts in understanding.

Example:

### v0.3

* Introduced coordination as core constraint
* Removed attention as primary bottleneck

---

# 🧱 Principles

## 1. One idea per change

Avoid mixing concepts

## 2. Clarity > complexity

Simple ideas scale better

## 3. Preserve evolution

Do not erase old thinking—improve it

## 4. Be comfortable being wrong

Progress = visible correction

---

# 🚀 What This Becomes

Over time, this repository becomes:

* a structured theory
* a history of thinking
* a system for generating new insights

---

# ⚡ Final Note

This is not a notes repository.

It is a **version-controlled thinking system**.
